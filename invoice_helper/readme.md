# 發票湊數小幫手 (Invoice Helper)

這是一個基於 Python 的工具，用於解決「子集合加總問題」（Subset Sum Problem）。它可以協助使用者從一堆發票金額中，找出總和最接近目標金額的組合。

## 專案結構與重構說明

本專案經過重構，將核心邏輯抽離，並提供兩種演算法以供比較。

### 檔案列表

* **`price.txt`**: 輸入資料檔。第一行為目標金額，其後為可用數字。
* **`invoice_lib.py`**: 核心函式庫。
    * 包含 **動態規劃 (Dynamic Programming)**：高效，適用於大多數情況。
    * 包含 **暴力列舉 (Brute Force)**：精確但在數據量大時極慢，用於驗證 DP 的正確性。
* **`invoice_cli.py`**: 命令行工具。
    * 執行後會同時運行兩種演算法（若數據量允許），比較其結果與執行時間。
* **`invoice_gui.pyw`**: 圖形介面工具。
    * 使用 Tkinter 介面，預設使用高效的 DP 演算法。

### 演算法比較

| 特性 | 動態規劃 (DP) | 暴力列舉 (Brute Force) |
| :--- | :--- | :--- |
| **時間複雜度** | $O(N \times Target)$ | $O(2^N)$ |
| **適用場景** | 數字多、目標金額整數時 | 數字少 ($N < 25$)、需絕對精確驗證時 |
| **優點** | 速度極快，能處理大量數據 | 邏輯直觀，必能找出所有組合 |
| **缺點** | 僅適用於整數，記憶體消耗隨 Target 增加 | 隨 $N$ 增加，時間呈指數爆炸 |
